mongoose

mongoose是什么？

Mongoose是MongoDB的一个对象模型工具 同时它也是针对MongoDB操作的一个对象模型库,封装了MongoDB对文档的的一些增删改查等常用方法 让NodeJS操作Mongodb数据库变得更加灵活简单

安装mongoose

使用mongoose

Schema

  • Schema是数据库集合的模型骨架 定义了集合中的字段的名称和类型以及默认值等信息；也就是数据库里的内容，它不能操作数据库，和数据库无关只定义字段和类型,它是mongoose的一个属性
  • Schema生成Model，Model创造Entity

    定义Schema

Model

  • Model是由通过Schema构造而成 依赖于Schema 可以操作数据库 除了具有Schema定义的数据库骨架以外，还可以操作数据库 如何通过Schema来创建Model呢
  • 拥有了Model，我们也就拥有了操作数据库的能力

在数据库里查看的是复数 在数据库中的集合名称等于 模型名转小写再转复数,比如 User>user>users,Child>child>children

Mongodb

Mongodb 数据库

1.什么是MongoDB

  • MongoDB是一个基于分布式文件存储的开源数据库系统
  • MongoDB 将数据存储为一个文档，数据结构由键值对(key=>value)组成。MongoDB 文档类似于 JSON 对象。字段值可以包含其他文档，数组及文档数组。
  • 是前端最佳的数据库，里面可以写js代码，开源免费，单机性能非常高每秒能读写1.6万次
  • 关系查询功能不够完善
  • 在node里用mongodb，就是mongoose

3.mongodb启动与连接 下面2个都得启动

3.1 windows启动服务器端

  • 找到mongodb安装目录,一般是 C:\Program Files\MongoDB 2.6 Standard\bin
  • 按下Shift+鼠标右键,选择在此处打开命令窗口(就是DOM命令窗口)
  • 在除C盘外的盘符新建一个空目录,如 D:\Mongodb\data

  • 在打开的命令行中输入mongod –dbpath=刚创建的空目录,如mongod –dbpath=D:\Mongodb\data 再按回车键

  • .如果出现waiting for connections on port 27017就表示启动成功,已经在27017端口上监听了客户端的请求

  • 注意：–dbpath后的值表示数据库文件的存储路径,而且后面的路径必须事先创建好，必须已经存在，否则服务开启失败，每个操作数据库的时候都得启动服务器
  • 注意：这个命令窗体绝对不能关,关闭这个窗口就相当于停止了mongodb服务

3.2 启动客户端连接服务器

  • 找到mongodb安装目录,一般是 C:\Program Files\MongoDB 2.6 Standard\bin
  • 按下Shift+鼠标右键,选择在此处打开命令窗口 ，就是和上面的路径一样，在重新打开一个
  • 命令窗体中输入 mongo –host=127.0.0.1 或者 mongo 按回车键

4.MongoDB基本概念

  • 数据库 MongoDB的单个实例可以容纳多个独立的数据库,比如一个学生管理系统就可以对应一个数据库实例 Database
  • 集合 数据库是由集合组成的,一个集合用来表示一个实体,如学生集合 Collections:内容都在这里
  • 文档 集合是由文档组成的，一个文档表示一条记录,比如一位同学张三就是一个文档,也就是里面的内容
  1. 数据库操作
  • 切换数据库，切换过来以后此数据库并不会创建

注：如果此数据库存在，则切换到此数据库下,如果此数据库还不存在也可以切过来

  • 插入文档 当你向数据库的某个集合下面插入一个文档的时候，数据库和集合才会创建

db是代表当前数据库的名字，只输入一个db的时候会查看当前的数据库；
persons是自定义的集合名称；如果有这个数据库就会向里面插入，没有会自动创建这个数据库

  • 查看当前服务中有多少数据库

5.1 主键

当我们保存一个文档的时候，如果没有提供_id属性，mongodb会帮我们自动生成一个_id.这就是主键
1.它与业务无关
2.每个文档的主键不能相同，也不能为空，就像身份证号一样
3.无论文档的属性如何改变，主键不能修改，就像年龄一样，怎么变这个身份证号也不会变
4.只要知道了这个主键，就能找到其它的属性
5.主键能唯一标识一个文档

5.2 删除数据库,代表删除当前数据库

  1. 集合的操作

  • 查看当前数据库下有多少个集合

  • 删除某个集合

  • 创建一个空集合

_id:1 可以自己指定id,不然系统会自动生成一个,这就是要插入的文档;每当插入一条新文档的时候mongodb会自动为此文档生成一个_id属性,_id一定是唯一的，用来唯一标识一个文档 _id也可以直接指定，但如果数据库中此集合下已经有此_id的话插入会失败

  • 修改集合中插入的内容(也是字段)save()

注：如果不指定 _id 字段 save() 方法类似于 insert() 方法。如果指定 _id 字段，则会更新该 _id 的数据。就是里面有字段就是修改没有就是插入

  • 查询项目中的所有文档，也就是看一个数据库有几条内容
  1. 更新文档
  • query(_id:1) 查询条件,指定要更新符合哪些条件的文档
  • upsert 可选，这个参数的意思是，如果不存在符合条件的记录时是否插入updateObj. 默认是false,不插入。
    如果upsert=ture代表找到就修改，没有就插入
  • multi 可选，mongodb 默认只更新找到的第一条记录，如果这个参数为true,就更新所有符合条件的记录。

实例

  • 递增,如果以前的age是55，那么现在就是60，$inc在原基础上累加

注：如果有多条_id是1的数据只更新一条,如果想全部更新需要指定{multi:true}的参数

angular

angular怎么用

  • 处理数据，前后端分离的框架，双向数据绑定，扩展标签(自定义指令)的功能，模块化，单页开发(头尾不动，只动中间)

    ng-app

  • 启动angular程序，指定模块来启动，定义模块。调用run方法，产生$rootScope根作用域

    ng-model

  • 只能放变量，不能放运算
  • 用于表单元素
  • 如果作用域上没有，则声明，如果有，则取值放到输入框中
  • 如果使用user.name 则会创建一个对象，将name变成一个变量存入进来

    ng-bind

  • ng-bind不会闪烁
  • 不能用于表单元素
  • 可以使用运算符 ! / ? :
  • 原有的内容会被ng-bind绑定的值替换掉

    ng-cloak

    用途：防止闪烁
  • 先声明一个css，找到所有带有ng-cloak让其属性
  • 增加到需要防止闪烁的元素身上

  • 加载后移除掉ng-cloak属性

    ng-controller

    用途：调度数据的，买一送一送$scope作用域

  • 不能操作dom，DOM在link函数中操作

  • 管辖范围，和dom平齐
  • 可以将带有ng-controller的标签嵌套，子继承父，父继承不了子
  • 不能复用
  • 调用ng-controller后可以窒息感控制器

ng-repeat

用途：循环遍历，迭代。

  • 要循环谁，就写在谁的身上
  • 遍历数组，ng中增加的属性$index $odd $even $first $last $middle(根据索引来遍历)
  • 通过(key,value)可以获得索引，默认写一个变量则是value
  • 对象同数组
  • 为什么加track by $index? 如果value重复，不能通过相同的value去遍历，通过唯一项来进行遍历。只要是数组就加track by $index

    ng-click

  • angular的事件 ng-click=”a()”

ng-show/ng-hide

用途：控制元素的显示与隐藏

  • 通过css样式进行隐藏，内部命令依然执行

频繁切换采用ng-show/ng-hide

ng-if

用途：控制元素的显示与隐藏

  • 通过删除DOM达到隐藏效果，如果为false，内部指令不再执行
  • 会形成一个作用域，保留删掉的内容

一般和ng-repeat连用，如果数据不存在就移除掉dom

ng-disabled

用途：ng-disabled=”true” 进入禁用状态

ng-readonly

用途：ng-readonly=”true” 进入仅读状态

指令(内置，自定义)

  • 自定义:组件式和装饰型
    • 组件式：Element，看有没有template 目的是复用
    • 装饰型：Attribute，一般没有模板，link函数
